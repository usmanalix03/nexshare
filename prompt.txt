4. Feature Specs: Mode 1 - "Lightning Share" (Pure WebRTC P2P)
This mode bypasses the database entirely for instant, local transfers.

The Engine: Use the browser's native WebRTC API to create a direct peer-to-peer data channel. Files must automatically trigger a download on the receiving end.

The Matchmaker (Socket.io): The server acts only as a signaling server to pass SDP offers/answers and ICE candidates.

Hybrid Discovery Logic:

Radar Mode (Automatic): The server reads the x-forwarded-for or IP of incoming connections. Clients with the identical public IP are grouped into a hidden Socket.io room and instantly appear on each other's UI "Radar".

Room Code (Manual): Provide a UI fallback to generate a 5-digit code. Clients entering the same code join a specific Socket.io room to share across different networks.

5. Feature Specs: Mode 2 - "Secure Vault" (Cloud + AES + DB)
This mode is for asynchronous sharing via expiring links.

Upload & Encrypt: The user uploads a file. The Express backend uses multer to receive it and streams it through Node's crypto module using AES-256-CBC before saving it to the server disk.

Database (CockroachDB): Create a Model that stores: id, original_filename, encrypted_filepath, iv (Initialization Vector), encryption_key, created_at, and expires_at (24-hour TTL).

Link Generation: Return a unique, secure URL (e.g., /vault/[uuid]) to the frontend.

Download & Decrypt: When the receiver visits the URL, the controller validates the TTL in CockroachDB. If valid, it fetches the file, decrypts it on the fly using the stored key/IV, and pipes it to the response stream.
